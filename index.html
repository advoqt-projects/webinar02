Web Application Hacking Webinar

A CyberWarrior Webinar

"I have no idea what I'm doing"

Do you know what a web application is? Have you ever used one?

A web application is simply an application that runs on a web browser, so you need a WEB BROWSER to use them.

Youtube, Facebook, Twitter, Gmail are all examples of

WEB APPLICATIONS !

and you know what?, THEY CAN BE HACKED! er.. we will talk about that later...

Web applications use the Server Client model, the server provides a service for the client, the server is the burger guy and the client is, well the client of the burger shop.

In the case of web applications, the client is the application running on the web browser, which consumes the services offered by the server, one of them being the database

Ok, now that you know how web applications work, let's learn the different techniques used to hack them!

Every web application is insecure to some degree, hackers exploit their vulnerabilities to steal sensitive data, cause havoc and other evil deeds!

Hackers love to steal passwords, but how they do that?

They do it using... BRUTE FORCE!

Brute force means that hackers test every possible password until they find the correct one, It gets progressively more difficult as the target password grows in size and uses a more varied pool of characters

Don't use common words like 'candy' as passwords, or stealing passwords will be just as easy as stealing a candy from a baby!

Now you know the importance of using really long passwords, huh?

The next technique we will talk about is called SQL injection. What is SQL?, why injection?, those are easy ones... SQL is the language used to communicate with databases, the English of Databases!

What does it mean to inject it though?, Think of it this way, "Eat your dinner" and "Eat, you're dinner" are not the same, right?, we "injected" some characters into the second phrase, and now the meaning of the phrases are totally different!, it's the same for the English of databases, er.. I mean SQL.

SQL Injection is used to alter the original objective of a SQL query, to perform evil deeds like avoiding password checks, steal data, remove data and creating new data!

Whew, let's move on to the next technique, called Cross Site Scripting or XSS, what a mouthful right?

XSS is mostly used to steal sessions, what is a session you ask?

A session is like a train ticket, you have it with you during a train ride, it proves that you are a rightful passenger. Well imagine what could happen if someone stole your ticket...

With the ticket in hand (your name is on there!) someone could impersonate you, and do some evil deeds on your behalf!

Cross Site Scripting is a way for hackers to do evil stuff on behalf of other users without even having to know their passwords!

Ok, enough of that, let's go Phishing!!

Yes, you heard it right, Phishing, with PH, and no, I don't need to go back to elementary school.

Phishing is another hacking technique!

How does that work?

Well, imagine that your favorite flavor of ice cream is strawberry, and you don't eat anything else (you get sick)

Your friend wants to prank you by painting a vanilla ice cream red to make it look like the real thing. Sadly you fall for it!

You are now a victim of phishing, poor you!

Phishing is used, for example, to fake bank web applications so that users willingly give their passwords thinking it's the real deal! The links to these fakes are mostly sent by email or shared on social media!

That's why you need to always, ALWAYS, check that you are accessing the correct web application (look at the address!)
